‚ùå Potential Issues to Look For

If you see this in console:

üîç Available fields in energyData: ["pv_p", "battery_p", "grid_p"]  // load_p missing!

Solution: Backend is not including load_p in energy_chart data

If you see this:

‚ùå Load mapping 0: No load_p data found in either source

Solution: load_p is missing from both energyData and batteryPriceData

If you see this:

Table entry 0: load_p=undefined, exists=false

Solution: Confirms load_p field is not being sent from backend

üõ† Backend Fix Required

If the debug logs show that load_p is missing from energyData, the issue is in the backend DownloadController.php:

Check line ~422 in DownloadController.php:

// Energy chart data - should include load_p
$result['energy_chart'][$timestamp] = [
    'pv_p' => $snapshot->pv_p ?? 0,
    'battery_p' => $snapshot->battery_p ?? 0,
    'grid_p' => $snapshot->grid_p ?? 0,
    'load_p' => $snapshot->load_p ?? 0  // ‚Üê This line must be present!
];

If this line is missing or commented out, that's the problem!
